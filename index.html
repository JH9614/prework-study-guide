HTML

  • The head element contains informatino about the webpage (meta data).
  • The body element represents the visible content shown to the user (semantic code).
  • "Element" refers to the entire element. For example, from the opening to the closing of a semantic element.
  • "Tag" refers to the letters or numbers held within the element brackets i.e. the "li" for the "list item" element.

HTML Elements

  • "div" is the tag for the element that is used to create a container to group content so that it can be easily styled with CSS. It does nothing semantic until it is styled. Here's a link to an example of its use.
  • "a" is the tag for the "anchor" element. This element has many attributes that may be used with it, a link to a list of them is in the "href" list item. When you click the link, just scroll down.
  • "table" is the tag that is used to create a table feature in the web page. It has multiple distinct sub tags related to it. Two of those being the "thead", "/thead" and the "tbody", "/tbody". These are used to create the head and body of the table feature respectively. Sub to the previous two are the "tr", "/tr" the "th", "/th" and the "td", "/td". The "tr" is to create each individual row for the table. The "th" is to create the headding for each column in the table. And finally "td" is to create the spot where the actual data goes (it stands for "table data").

HTML Attributes

  • GLOBAL ATTRIBUTES: These are attributes that can be used with all HTML element. Any element in this list will have a (GA) by the statement of its tag to show that it is one.
  • "href" is the tag for an attribute that is generally only used with the "anchor" element. The href attribute allows a link to be connected to a bit of text or an image, etc. that is contained within the "anchor" element. Here is a guide to its use and some examples.

      • CSS

      • CSS stands for "Cascading Style Sheets" and is what is used to decorate a webpage
      • CSS elements don't require any type of bracket around them.
      • To use CSS simply state the tag (what's inside an HTML element's brackets) of the HTML element and any attributes added to that CSS element will be applied to the HTML element. Any attributes added to a CSS element will be placed with in the confines of a set of curly brackets/braces ie. these things: {}.
      • Attributes in CSS can be applied to multiple different CSS elements at once. This is done by stating an element, adding a comma, then stating the next element. This can be done with infinitely many CSS elements. When the comma(s) has been added beween the CSS elements, the attribute(s) to be applied to them can be written like normal (like for a single element). Be sure to place the separate CSS elements on different line (ie. row 68, then 69) to aid in legibility.
      • A margin indicates how much space we want around the OUTSIDE of an element.
      • Padding indicates how much space we want around the content INSIDE an element.
      • A collection of CSS resources for beginners.

      CSS Elements (Called Rules)

      • CSS elements are called "rules".
      • The CSS selector is what could be considered the equivolent of an HTML tag. It is the declarative item that starts the "rule". For example, to apply color to an HTML heading of size one (h1), the "rule" starts with the CSS selector "h1".
      • "*" means "all elements". It applies its self to all semantic (means to be seen) HTML elements on an HTML document. So if there are any attributes within this CSS element, those attributes will be applied to every HTML element in the HTML document.

      Git

      • Git commands are sets of commands for the CL (command line) of a computer's system. They are called git commands because "Git" is the software that is used to access the command line.
      • List of basic commands.
      • Another list of Git Bash commands for developers can be found here.
      • "git status": checks what branch we are currently on.
      • "git checkout -b branch name": creates a new branch and switches to it.
      • "git log": shows all commits that have been performed on a repo.

      Git Work Flow Commands In Order

      1. "git status": checks what branch we are currently on.
      2. "git add -A": stages ALL (hence the "A") the files that are yet to be staged.
      3. "git commit -m "added code to the HTML file" ": use this to commit (save) the files that have been staged to the working branch. The bit in quotes is the note describing what was done in the files.
      4. "git pull origin main": syncs the local branch you are working on with the remote branch of the same name on github. It is always important to do a pull before you "push" the code into the branch of the remote repository on github. If you don't sync the branches you may be working on outdated code.
      5. "git push origin "branch/name"": this "pushes" the changes we made to the remote branch so it has a copy of what the file looks like now, replacing the old one.

      JavaScript

      • JavaScript is a widely used language that is used by websites to handle multimedia, animated images, and create dynamically updating content.
      • JavaScript is also the code that links the front end to the back end (I think).
      • A variable is a named container that allows us to store data in our code.
      • Control flow refers to how the code we input is executed by the computer. For JS the control flow is top to bottom and for multi faceted variables, functions, objects, arrays and the like, it is executed left to right.
